# Offensive Security Intro — Writeup

Platform: TryHackMe  
Room: Offensive Security Intro  
Difficulty: Beginner  
Category: Offensive Security  

---

## Objective

This room introduces the fundamentals of **offensive security**, allowing beginners to understand how ethical hackers analyze systems and identify vulnerabilities in web applications.

The objective of this lab was to simulate a simple attack against a website in a safe and controlled learning environment.

---

## Task 1 — Thinking Like a Hacker

The first step introduces the **mindset of an ethical hacker**.

### Key Concept

Security professionals must think like attackers in order to identify weaknesses before malicious actors can exploit them.

### Main Learnings

- Ethical hacking is used to legally test systems for vulnerabilities  
- The goal is to improve the overall security of systems and applications  
- Understanding the attacker's perspective helps security professionals better defend systems  

---

## Task 2 — Starting the Lab

In this step, the lab environment was launched.

This allowed interaction with a simulated website that contains intentionally vulnerable components for educational purposes.

### Concepts Introduced

- Safe environments for learning cybersecurity  
- Practical hacking labs  
- Hands-on experience with vulnerable systems  

These types of environments are commonly used in cybersecurity training platforms.

---

## Task 3 — Finding Hidden Pages

In this stage, the objective was to discover **hidden pages within the website**.

This represents a common activity performed during the **reconnaissance phase** of penetration testing.

### Main Learnings

- Websites may contain hidden or non-indexed pages  
- Attackers often explore directories and endpoints to find sensitive areas  
- Discovering hidden resources can expose administrative or restricted functionality  

This step demonstrates how reconnaissance is an essential phase in security testing.

---

## Task 4 — Attacking the Admin Page

In the final step, the task involved interacting with an **administrative login page** and analyzing potential weaknesses in the authentication mechanism.

This activity demonstrates a basic scenario where attackers attempt to access restricted areas of a web application.

### Concepts Learned

- Authentication vulnerabilities  
- Importance of secure login mechanisms  
- Risks associated with poorly protected administrative pages  

Understanding these weaknesses helps security professionals design stronger authentication systems.

---

## Tools / Concepts Introduced

Although this was an introductory room, several important cybersecurity concepts were presented:

- Web application reconnaissance  
- Hidden page discovery  
- Authentication analysis  
- Ethical hacking mindset  

These concepts are commonly used during the early stages of penetration testing.

---

## Skills Developed

During this room, the following foundational skills were introduced:

- Ethical hacker mindset  
- Web application reconnaissance  
- Discovery of hidden web pages  
- Basic analysis of authentication mechanisms  

These skills are fundamental for individuals beginning their journey in **offensive cybersecurity**.

This lab represented my **first practical experience with offensive security concepts**.
